## Осознанно не сделано на Этапе 1

- Реальный инференс (llama.cpp/Metal) — только `MockInferenceEngine`-подобная
  симуляция прямо в `ChatView`, без выделенного протокола `LocalInferenceEngine`
  (появится в Этапе 3/5, чтобы не проектировать архитектуру раньше интерфейса,
  который её проверяет).
- Реальные системные разрешения (EventKit/Contacts/Photos) — только UI центра
  разрешений на моках.
- Реальная долговременная память (SwiftData + шифрование) — только UI на
  моковых записях в памяти процесса.
- Реальная загрузка моделей — экран загрузки анимирует прогресс таймером.

## История сборок CI

- **2026-07-13** — первая зелёная сборка `Build` (run
  [29276160422](https://github.com/MakrSas/Luma/actions/runs/29276160422),
  коммит `c4bdd33`). Артефакт `Luma-unsigned-ipa` (~288 КБ) доступен в
  Actions artifacts репозитория (срок хранения 30 дней). По пути до зелёной
  сборки исправлены три проблемы CI (см. историю коммитов и
  `KNOWN_ISSUES.md`): область видимости `RiskLevel` между таргетами
  приложения и виджета, лишний параметр `isEnabled` в вызове реального
  `glassEffect` (сигнатура Xcode 26.3 отличалась от предположенной), и имя
  симулятора для тестов (`iPhone 15` больше не в образе раннера — заменено
  на `iPhone 17`).
- **2026-07-13** — редизайн по прямой обратной связи пользователя со
  скриншотами официального приложения Siri (run
  [29278428291](https://github.com/MakrSas/Luma/actions/runs/29278428291),
  коммит `39f7c29`), артефакт `Luma-unsigned-ipa` (~298 КБ). Изменено:
  история — стартовый экран вместо чата; убран янтарный акцент, палитра
  строго монохромная (`LumaColor.accent`/`onAccent`); панель ввода чата
  переделана в единую капсулу «+ (меню модели/интеллекта) → поле → круглая
  кнопка отправки»; экран истории переработан под структуру референса
  (без заголовка/секций, плавающие кнопки фильтра/поиска/создания диалога);
  добавлены rich-карточки ответа (`RichAnswerCard`/`RichAnswerCardView`) —
  подробности в `DESIGN.md`. По пути исправлен тест, ссылавшийся на
  удалённое поле `AppState.isTemporaryChatActive`.
- **2026-07-13** — правка по обратной связи со скриншотами реальной сборки
  (run [29279796797](https://github.com/MakrSas/Luma/actions/runs/29279796797),
  коммит `76bf0d0`), артефакт `Luma-unsigned-ipa` (~299 КБ). Панель ввода
  чата разделена на три отдельных элемента («+», поле, кнопка отправки)
  вместо одной слитой капсулы, само поле увеличено; плавающие кнопки
  экрана истории увеличены (40/44 → 48/56pt) и добавлен отступ снизу, чтобы
  карточки не заезжали под них.
- **2026-07-13** — каталог native-виджетов ответа (run
  [29280897917](https://github.com/MakrSas/Luma/actions/runs/29280897917),
  коммит `d857cd2`), артефакт `Luma-unsigned-ipa` (~313 КБ). По референсу
  системного виджета «Аккумуляторы» и статус-строки Tesla/Tessie из
  скриншотов Siri: `AnswerWidget`/`AnswerWidgetView` рисуют результат
  инструмента без фона/обводки/стекла — как настоящий системный виджет;
  несколько плиток за один ответ агента складываются в сетку. Есть рабочий
  моковый триггер — вопрос про заряд/память в чате отвечает настоящим
  виджетом (или двумя). Подробности в `DESIGN.md`.
- **2026-07-13/14** — первый настоящий локальный инференс (run
  [29286198035](https://github.com/MakrSas/Luma/actions/runs/29286198035),
  коммит `828726e`), артефакт `Luma-unsigned-ipa` (~7.2 МБ — вырос из-за
  встроенного MLX/Metal рантайма). По запросу пользователя проверить
  виджеты «в реальности»: подключён `LocalLLMClient` (MLX-бэкенд,
  `Luma/Inference/LocalInferenceEngine.swift` + `MLXInferenceEngine.swift`).
  По пути обойдены три независимых апстрим-бага (подробности и ссылки на
  issue в `ARCHITECTURE.md` → «Реальный локальный инференс»): валидация
  Swift-макросов на headless CI, неверное имя SPM-продукта, сломанный
  llama.cpp-бэкенд пакета (перешли на MLX) и breaking change в свежем
  релизе `swift-jinja` (запинен). Каталог реальных моделей, реальная
  загрузка и подключение к `ChatView` — ещё не сделаны (следующий шаг).
- **2026-07-14** — вся цепочка «реальный ИИ» замкнута (run
  [29306947640](https://github.com/MakrSas/Luma/actions/runs/29306947640),
  коммит `a73cd08`), артефакт `Luma-unsigned-ipa` (~6.9 МБ). Каталог моделей
  теперь два настоящих репозитория `mlx-community` на HuggingFace
  (Qwen2.5-1.5B/0.5B-Instruct-4bit), `ModelDownloader` реально качает файлы
  и проверяет SHA-256 весов, `DeviceStatusProvider` читает настоящий заряд
  батареи и свободное место, `ChatView` генерирует ответ через настоящий
  `MLXInferenceEngine` на выбранной скачанной модели (без модели — честно
  просит её скачать, без выдуманного текста). Подробности в
  `ARCHITECTURE.md`, честные ограничения (частичный resume закачки,
  качественные, не измеренные оценки моделей) — в `KNOWN_ISSUES.md`.
- **2026-07-14** — настоящий tool-calling вместо ключевых слов (run
  [29313000876](https://github.com/MakrSas/Luma/actions/runs/29313000876),
  коммит `4ab4cc8`), артефакт `Luma-unsigned-ipa` (~6.9 МБ). По прямому
  запросу пользователя убрать сопоставление по ключевым словам:
  `Luma/Inference/DeviceTools.swift` — три настоящих `LLMTool`
  (`get_battery_status`/`get_storage_status`/`get_system_version`), модель
  сама решает, вызывать ли их, через структурированный function calling.
  `MLXInferenceEngine` реализует ручной агентный цикл поверх
  `responseStream`/`resumeStream` (библиотека не выполняет инструмент и не
  продолжает генерацию сама — только детектирует вызов). Заодно исправлены:
  нечитаемый белый текст на залитых кнопках (кастомный `ButtonStyle`),
  несовпадение ключевых слов с русскими склонениями (что теперь не
  актуально — эвристика убрана целиком), и поиск в истории перенесён вниз
  по нативному паттерну iOS 26 Liquid Glass.
