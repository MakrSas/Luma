# CLAUDE.md — постоянные правила проекта Luma

Этот файл читает Claude (или любой другой ассистент) в начале каждой новой сессии
по этому репозиторию. Здесь зафиксированы решения, которые не нужно пересматривать
заново, и правила, которые нельзя нарушать без явного запроса пользователя.

## Что такое Luma

Luma — нативное iOS-приложение: локальный текстовый ИИ-агент для iPhone.
Пользователь вводит текст, локальная модель (GGUF через llama.cpp/Metal, позже —
возможно MLX) понимает запрос, выбирает инструменты приложения и выполняет
разрешённые действия с подтверждением рискованных операций.

Название **Luma** — это только название приложения. Сам агент не должен называть
себя Luma, не должен писать «Я Luma» и не должен упоминать название по своей
инициативе — только если пользователь прямо спросит о самом приложении.

## Порядок чтения при новой сессии

1. Прочитать `STATUS.md` — что уже сделано и на каком этапе проект.
2. Прочитать `CURRENT_TASK.md` — что делать прямо сейчас.
3. Прочитать `DESIGN.md` — утверждённый визуальный язык, не проводить дизайн-аудит заново.
4. Прочитать `ARCHITECTURE.md` — структура кода и границы модулей.
5. Прочитать `KNOWN_ISSUES.md` — известные проблемы, чтобы не наступать на те же грабли.

## Жёсткие правила

- **Голосового ввода нет.** Ни кнопки микрофона, ни распознавания речи, ни синтеза
  речи, ни голосовой активации. Это осознанное решение, не забытая фича.
- **Liquid Glass — только настоящий SwiftUI API** (`glassEffect`, `GlassEffectContainer`,
  `.buttonStyle(.glass)` / `.glassProminent`), с fallback на `Material` через
  `#available(iOS 26.0, *)`. Никакой имитации стекла через голый `.blur()`.
  Основной контент никогда не находится целиком под стеклом — стекло только для
  чрома: ввод, навигация, пикеры, подтверждения, плавающие элементы.
- **Не копировать интерфейс Siri или ChatGPT.** Референсы (скриншоты новой Siri)
  использованы только как направление композиции — крупная типографика, свободная
  вёрстка, асимметричная сетка истории. У Luma собственная цветовая идентичность
  (тёплый янтарный акцент на графитовом фоне — см. `DESIGN.md`), не
  фиолетово-синяя AI-палитра.
- **Никаких приватных API**, обхода sandbox, чтения интерфейса других приложений,
  произвольных касаний в чужих приложениях.
- **Никакого WebView как основного интерфейса.**
- Модель никогда не выполняет произвольный код — только строго типизированные
  инструменты из `ToolRegistry` с явным описанием, уровнем риска и требуемыми
  разрешениями.
- Разработка идёт по этапам (см. `STATUS.md`). **Не перескакивать через этапы**
  без явного подтверждения пользователя после проверки на реальном iPhone.
- Пользователь собирает проект только через GitHub Actions — у него нет
  современного Mac. Любое изменение должно оставаться собираемым в
  unsigned-режиме без секретов (см. `.github/workflows/build.yml`).
- Не выдавать моковые данные за настоящую реализацию и не заявлять, что сборка
  прошла, пока GitHub Actions реально не завершился успешно.

## Технологии

Swift, SwiftUI, Swift Concurrency, Observation (`@Observable`), SwiftData (со
Stage 2), ActivityKit, WidgetKit, App Intents, EventKit, Photos, Contacts,
CryptoKit, Keychain, URLSession, BackgroundTasks, XcodeGen, Swift Package Manager.

Не использовать: Flutter, React Native, Storyboards.

## Этапы

См. `STATUS.md` за актуальным статусом. Этап 1 (дизайн + моковое приложение)
не подключает llama.cpp, реальные модели, настоящие разрешения системы и
настоящую память — это Этапы 2–7.
