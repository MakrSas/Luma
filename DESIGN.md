# DESIGN.md — визуальный язык Luma

Этот документ фиксирует утверждённые дизайн-решения Этапа 1. Не проводить
дизайн-аудит заново в новых сессиях — только дополнять по мере необходимости.

**2026-07-13: редизайн по прямой обратной связи пользователя со скриншотами
официального приложения Siri (iOS 27, светлая/тёмная история, экран
разговора с изображениями/картой, панель ввода).** Ниже — актуальные
решения. Более ранние версии этого файла описывали янтарный акцент и чат как
стартовый экран — оба решения отменены.

**2026-07-15: второй раунд «один в один» по расширенному набору референсов
(рендеры Bloomberg + скриншоты реальной беты iOS 27 + материалы Apple
Newsroom), по прямому запросу пользователя «строго следуй всем правилам
Apple без отхождений».** Ключевые изменения этого раунда:

- палитра стала **чисто нейтральной** (r == g == b) — тёплый подтон убран;
- тёмный канвас — **чистый чёрный** (`UIColor.black`), как фон разговора и
  истории в Siri; светлый — нейтральный `0.949`;
- пузырь пользователя в чате — **нейтральный серый** («received»-серый,
  `LumaColor.bubble`: `#E9E9EB` / `#2C2C2E`), а не заливка `accent`: в
  референсах вопрос пользователя — серый пузырь, ответ — плоский текст;
- карточки истории: радиус 24 (`LumaRadius.card`), паддинг `md`, заголовок
  `title3 semibold`, **без обводки** — только заливка `canvasElevated` +
  мягкая ambient-тень (читается в светлой теме, исчезает на чёрном);
- ~~панель ввода чата и нижние кнопки истории обёрнуты в
  `LumaGlass.container`~~ — **отменено в тот же день** (см. блок ниже):
  контейнер сплавляет соседние стеклянные формы в одну полосу-подложку;
- арт кастомных иконок нормализован: кольцо каждого варианта отмасштабировано
  до той же доли плитки (0.639 ширины), которую занимает глиф основной
  Icon Composer-иконки на реальном рендере iOS (замер по скриншоту
  симулятора, а не по CLI-превью — они расходятся).

**2026-07-15 (поздний вечер): четвёртый раунд по скриншотам реальной
сборки.**

- **Панель ввода чата — полностью непрозрачная** (`canvasElevated` круги
  44pt и капсула, как в референсе iOS 27 Siri). Стекло убрано и отсюда:
  каждая glass-вью рендерит собственную backdrop-область, и объединение
  трёх таких областей выглядело как полоса-подложка на всю ширину — тот
  самый артефакт, о котором пользователь сообщал трижды. Непрозрачные
  заливки исключают его физически.
- **`ToolActionCardView` удалена**: карточки статуса инструмента
  («Напоминание создано · create_reminder») в чате больше не показываются
  (`role == .toolAction` отфильтрован при загрузке) — итог несёт текст
  ответа ассистента.
- **`RichAnswerCardView` без фона**: иконка-плашка + текст прямо на
  канвасе.
- **Кнопки действий на экранах модели** (`Скачать/Удалить/Отменить`) —
  `.controlSize(.large)`, была тонкая полоска.
- **Иконка переведена с Icon Composer `.icon` на классический
  `.appiconset`** (light/dark/tinted PNG): формат `.icon` поддерживает
  специализации только default/dark/tinted, а Clear-режим для слоёных
  иконок система всегда рендерит стеклянным барельефом. Пользователь
  хочет в Clear сплошной белый глиф «как у Grok» — Grok это классическая
  иконка, для которых iOS строит Clear из tinted-варианта. Глиф во всех
  трёх PNG — 654/1024 ширины (замеренная доля утверждённого рендера).

**2026-07-15 (вечер): третий раунд по скриншотам реальной сборки.**

- **Не оборачивать панель ввода чата и нижние кнопки истории в
  `GlassEffectContainer`**: при малых зазорах контейнер сливает стеклянные
  формы в одну полосу на всю ширину — она и есть та «подложка под строкой
  ввода», на которую пользователь жаловался уже дважды. Контейнер
  зарезервирован для случаев, где слияние — желаемый эффект.
- **Карточки в чате — непрозрачные**: `RichAnswerCardView` и
  `ToolActionCardView` теперь `canvasElevated`-заливка без стекла (как
  непрозрачные тёмно-серые карточки новостей в референсе Siri). Стекло в
  чате остаётся только у чрома: панель ввода, карточка подтверждения
  рискованного действия, статус-чип.
- **Размер круглых кнопок задаётся `.controlSize(.large)`**, не `.frame()`:
  рамка вокруг стеклянной кнопки только расширяет layout-бокс, а видимый
  круг продолжает обтягивать label. Плюс явный размер иконки
  (~17–19pt medium). Это третья итерация по жалобе «кнопки маленькие» —
  предыдущие два подхода (константа 48pt, голый `.frame(44)`) не работали.
- **Функция кастомных иконок удалена целиком** по решению пользователя
  (пикер, ассеты, маршрут, `ASSETCATALOG_COMPILER_ALTERNATE_APPICON_NAMES`).
- **Иконка обновлена** новым артом «Плоская.png» (1502², прозрачный фон) →
  `foreground.png` 1024². У слоя `glass: false` и добавлена
  mono-специализация `tinted: белый` — чтобы в режиме Clear глиф рендерился
  просто белым (как у иконки Grok с референс-скриншота), а не стеклянным барельефом.
  Известный пробел CLI-превью: заливки специализаций оно не применяет
  (тёмное превью всегда показывает чёрный глиф) — проверять только на
  устройстве.

## Референсы

Пользователь предоставил конкретные скриншоты официального приложения Siri
(iOS 27) и явно указал ориентироваться на них буквально, а не изобретать
свою композицию:

- сетка истории: карточки без общего заголовка экрана, без разделения на
  секции «закреплённые/все» — только маленькая строка «дата + булавка» внутри
  каждой карточки; плавающая кнопка-фильтр (`line.3.horizontal.decrease`)
  в правом верхнем углу поверх контента; плавающие круглые кнопки поиска
  (лупа, слева) и создания диалога (карандаш, справа) внизу поверх контента;
  навигационного бара с заголовком нет вообще;
  каждая image-карточка — фото на весь блок с overlay-текстом (дата сверху,
  заголовок снизу на градиентной подложке);
- экран разговора: сообщения пользователя и ассистента, поверх ответа могут
  быть встроены иллюстрированные карточки-виджеты (например «Cricket Ball
  Details», карта с точками) — не просто текст;
- панель ввода: единая скруглённая капсула, слева простая иконка «+» без
  фонового кружка, по центру placeholder на всю ширину, справа — компактная
  круглая кнопка (микрофон в оригинале; у Luma это кнопка отправки/остановки).

Не копировать пиксель в пиксель, не использовать логотипы или название Siri,
не имитировать приватные жесты или анимации Apple. Но структуру экрана —
расположение элементов чрома, форму карточек, панель ввода — воспроизводить
точно, не придумывая альтернативную компоновку.

## Идентичность Luma

**Строго монохромная палитра.** Никакого брендового акцентного цвета —
пользователь явно попросил убрать янтарь и оставить только чёрный и белый.

- `LumaColor.accent` — чёрный в светлой теме, белый в тёмной («чернила» на
  фоне). Используется для интерактивных элементов, выделенных состояний,
  залитых кнопок.
- `LumaColor.onAccent` — обратный цвет для контраста поверх заливки
  `accent` (белый текст на чёрной кнопке в светлой теме и наоборот).
- Единственное исключение — семантические цвета риска
  (`LumaColor.success/warning/danger`, через `LumaColor.risk(_:)`): они несут
  функциональную информацию о безопасности действия в карточках
  подтверждения, и остаются собственных цветов — так же, как на референсе
  Siri зелёное кольцо индикатора заряда машины остаётся цветным в
  монохромном интерфейсе.
- **Канвас**: чисто нейтральный — чёрный (`UIColor.black`) в тёмной теме,
  светло-серый `0.949` в светлой. Карточки — `canvasElevated` (`#1C1C1E` /
  белый), пузырь пользователя — `LumaColor.bubble`.
- Цвета определены в `Luma/DesignSystem/LumaColor.swift`, поддерживают светлую
  и тёмную тему через `UIColor` с динамическим провайдером.

## Типографика

`Luma/DesignSystem/LumaTypography.swift` — без изменений в этом раунде.

- Крупные заголовки (`LumaType.display`) — системный serif, полужирный.
  Используется в пустом состоянии чата и заголовках detail-экранов. **Не** на
  экране истории — референс не показывает большого заголовка там.
- Остальной текст — системный шрифт, Dynamic Type-совместимый.
- Моноширинный шрифт — только для технических деталей.

## Liquid Glass — правила использования

`Luma/DesignSystem/LumaGlass.swift` — единая точка входа, без изменений
API. `lumaGlassProminentButtonStyle()` теперь явно применяет
`.tint(LumaColor.accent)`, чтобы не унаследовать системный синий.

Где используется стекло:
- элементы панели ввода в чате (кнопка «+», капсула поля, кнопка отправки —
  раздельные, без общего контейнера);
- плавающие кнопки истории (фильтр, поиск, создание диалога);
- шторки выбора модели и интеллекта;
- карточка подтверждения рискованного действия и статус-чип агента.

Карточки контента в чате (`RichAnswerCardView`, `ToolActionCardView`) — не
стекло, а непрозрачная заливка `canvasElevated` (2026-07-15, вечер).

Где стекло не используется: под списком сообщений/карточек целиком; карточки
истории (непрозрачные `canvasElevated` с тонкой обводкой — фото-карточки
непрозрачны по построению, это фото); Form/List в служебных настройках.

## Запрещённые визуальные решения

- Одинаковые прямоугольные карточки на каждом экране.
- Толстые цветные градиенты «ради AI-стиля».
- Любой брендовый акцентный цвет, кроме монохрома (см. выше про risk-цвета).
- Стекло поверх всего экрана.
- Огромные декоративные заголовки без функции (в т.ч. заголовок «История» —
  референс его не показывает).
- Кастомные контролы, которые выглядят менее нативно, чем стандартные.
- WebView. Имитация Liquid Glass обычным blur при доступном настоящем API.
- Кнопка/иконка микрофона где бы то ни было.
- Пустые «заглушки» вместо реального мокового контента: если экран
  показывает виджет/карточку, она должна рендерить настоящий (пусть и
  моковый) заголовок/текст/иллюстрацию, а не пустой прямоугольник или
  подпись «скоро здесь будет...».

## Компоновка и отступы

`Luma/DesignSystem/LumaSpacing.swift` — без изменений.

## Навигация: история — стартовый экран

**Изменение этого раунда.** Раньше стартовым экраном был чат; теперь по
прямому запросу пользователя — история (`HistoryView`), в точности как в
референсе Siri. `RootView`'s `NavigationStack` открывается сразу в
`HistoryView`. Чат (`ChatView`) — всегда экран, на который переходят
(из карточки истории или из плавающей кнопки создания диалога), никогда не
корень; поэтому кнопка «назад» в чате — стандартная системная стрелка
`NavigationStack`, отдельная кнопка-«галерея» в тулбаре чата не нужна и
удалена.

Создание нового/временного диалога происходит с экрана истории (плавающая
кнопка-карандаш внизу справа, меню из двух пунктов) — `AppState` хранит
временный диалог отдельно (`temporaryConversation`), не добавляя его в
основной список `conversations`, как и раньше описано в исходном ТЗ.

## Панель ввода чата

**Три отдельных элемента, не одна слитая капсула** (первая версия этого
раунда сливала «+», поле и кнопку в один `glassSurface` — по обратной связи
с реальных скриншотов сборки это выглядело как «слишком маленькая полоска»,
разделено обратно):

`(+)  [  текстовое поле, отдельная капсула  ]  (кнопка отправки/стоп)`

- «+» — отдельный круг 44×44 со своим `glassSurface`, открывает `Menu` с
  выбором модели и режима интеллекта (то, что раньше показывалось отдельным
  рядом чипов над полем ввода — убрано из ряда и перенесено в «+», чтобы не
  занимать лишнюю строку). Микрофона нет и не будет.
- Текстовое поле — своя капсула `glassSurface`, высота задаётся вертикальным
  паддингом (`LumaSpacing.sm` сверху/снизу), а не `.frame(minHeight:)`.
  **Важно**: `.frame(minHeight:)` напрямую на многострочном `TextField(axis:
  .vertical)` ломает позицию курсора (курсор рисуется не там, где текст,
  разрыв между строкой ввода и курсором) — это баг, воспроизведённый на
  реальной сборке. Высота поля должна задаваться только паддингом, как в
  этом коде и как это делает, например, ChatGPT.
- Кнопка отправки — отдельный круг 44×44, залитый `LumaColor.accent`, когда
  есть текст для отправки или идёт генерация (тогда иконка `stop.fill`);
  иначе полупрозрачная нейтральная заливка.
- Все три элемента обёрнуты в `LumaGlass.container` (`GlassEffectContainer`),
  чтобы на iOS 26+ они могли морфиться друг с другом при системных
  анимациях, оставаясь визуально раздельными в состоянии покоя.

## Каталог native-виджетов ответа

**Новое в этом раунде.** По обратной связи пользователя со скриншотом
системного виджета «Аккумуляторы» (Battery) и виджета статуса Tesla/Tessie
из референса Siri: агент может показать структурированный результат
(например, `get_device_status`) как **настоящий системный виджет**, а не как
карточку с фоном/обводкой. `Luma/Models/AnswerWidget.swift` +
`Luma/Screens/Chat/AnswerWidgetView.swift`:

- `AnswerWidgetKind.compactMetric` — один показатель целиком ответ:
  кольцо-индикатор с иконкой рядом с крупным значением, в одну строку. Для
  случая «в ответе один виджет» — по правке пользователя над мокапом
  (`Когда один виджет`).
- `AnswerWidgetKind.squareTile` — квадратная плитка: кольцо-индикатор с
  иконкой сверху, крупное значение снизу, **без подписи** (значение
  подписи несёт иконка + текст ответа рядом, не отдельная строка на
  плитке — по тому же мокапу, `Когда два`). **Композируется**: если агент
  возвращает несколько плиток за один ход, `AnswerWidgetGridView`
  раскладывает их равной 2-колоночной сеткой (`LazyVGrid` с
  `.flexible()`-колонками), а не одну под другой.
- `AnswerWidgetKind.row` — строка на всю ширину: маленькое кольцо-иконка +
  подпись + значение (опционально «· деталь») — как строка статуса
  подключённого приложения (машина, умный дом); здесь подпись остаётся,
  так как это единственный видимый идентификатор источника (имя машины).
- **Никакого фона, обводки или стекла** — виджет рисуется прямо на канвасе,
  как настоящий системный виджет, а не как ещё одна карточка.
- **Цвет кольца — правило настоящего виджета Battery, не «чем больше
  заряд, тем зеленее»**: монохромный (`LumaColor.textPrimary`) по
  умолчанию, зелёный **только** при активной зарядке (тогда же появляется
  бейдж `bolt.fill`), красный только при критически низком уровне. Раньше
  цвет считался от процента заряда, из-за чего кольцо было зелёным даже
  без бейджа молнии — это несоответствие исправлено.
- Сообщение с ролью `.widgets` (`ChatMessage.widgets: [AnswerWidget]?`)
  хранит массив — это и есть «композиция из нескольких виджетов в один ход».
- **Больше не сопоставление по ключевым словам.** Первая версия виджетов
  триггерилась по подстрокам в сообщении пользователя (`DeviceIntent`) — по
  прямому запросу пользователя («сделай так, чтобы додумывался сам, а не по
  шаблону») это заменено на настоящий вызов функций: три `LLMTool`
  (`Luma/Inference/DeviceTools.swift`) описаны в схеме, которую видит
  модель, и она сама решает, нужен ли ей `get_battery_status`/
  `get_storage_status`/`get_system_version`, прежде чем отвечать — так же,
  как решил бы вызвать инструмент любой другой function-calling агент.
  `DeviceToolWidgets` в `ChatView.swift` строит виджет по **имени
  фактически вызванного инструмента**, не по тексту вопроса. Числа в
  виджете — свежее чтение `DeviceStatusProvider`, а текст ответа —
  собственные слова модели, увидевшей результат того же инструмента.
  Раньше эта эвристика ловила подстроку «память», которая не входит в
  «памяти» из-за русского склонения — с реальным function-calling это
  ограничение исчезло вместе с самой эвристикой.

## Rich-карточки ответа («виджеты»)

**Новое в этом раунде**, по референсу Siri (карточка «Cricket Ball Details»
и подобные). `RichAnswerCard` + `RichAnswerCardView`
(`Luma/Screens/Chat/RichAnswerCardView.swift`) — иллюстрированная карточка
(SF Symbol на монохромной плашке + заголовок + описание + источник),
которую агент может прикрепить к ответу как отдельное сообщение с ролью
`.richCard`. Есть один настоящий моковый пример в диалоге «Отпуск в Мехико»
(карточка о Национальном музее антропологии) — не пустая заглушка.

## Экран истории

**Полностью переработан в этом раунде** под структуру референса:

- масонри-сетка (`MasonryLayout`, без изменений) без заголовка экрана и без
  секций «закреплённые/все» — сортировка «закреплённые сначала», остальное
  по дате (переключается через меню фильтра);
- у каждой карточки — строка меты (иконка булавки, если закреплена, + дата:
  время сегодня, «Вчера», день недели за последнюю неделю, иначе короткая
  дата — `metaLabel(for:)` в `HistoryView.swift`);
- фото-карточки (`heroImageName` задан) — SF Symbol на монохромной
  градиентной подложке во весь блок карточки, дата сверху и заголовок снизу
  наложены поверх с затемняющим градиентом для контраста — не отдельный блок
  над/под фото;
- поверх контента плавающая кнопка-фильтр (сортировка + пункт «Настройки» —
  единственная точка входа в `SettingsHubView` с этого экрана, так как
  референс не показывает отдельной иконки настроек);
- поиск живёт **внизу постоянно**, не за иконкой, открывающей поле сверху —
  по нативному паттерну iOS 26 Liquid Glass (поиск в таб-баре/нижней панели,
  а не отдельным слоем сверху экрана). Компактная капсула «Поиск» слева
  снизу разворачивается на всю ширину по тапу, отодвигая кнопку создания
  диалога, а не открывает отдельный оверлей — то же семейство анимации
  «элементы стекла сливаются», что и в панели ввода чата
  (`LumaGlass.container`).

## Контраст текста на залитых кнопках

Системные стили `.buttonStyle(.glassProminent)`/`.borderedProminent` сами
решают, каким цветом рисовать текст поверх заливки — и с монохромным
`LumaColor.accent` (белый в тёмной теме) иногда рисовали **белый текст на
белой кнопке**, нечитаемо. Заменено на `LumaProminentButtonStyle`
(`Luma/DesignSystem/LumaGlass.swift`) — самодельный `ButtonStyle`, который
всегда явно красит текст в `LumaColor.onAccent`, без надежды на
автоматический контраст системы.

## Утверждённые экраны (Этап 1)

Все 15 экранов из ТЗ реализованы на моковых данных: история (стартовый
экран), главный чат/конкретный разговор (`ChatView` с обязательным
`conversationID`), новый/временный диалог, каталог моделей, карточка/страница
модели, загрузка модели, настройки интеллекта, центр разрешений, память,
редактор памяти, журнал действий, настройки производительности,
диагностика, лицензии моделей.

`SettingsHubView` — дополнительный навигационный хаб (не входит в список 15),
точка входа — меню фильтра на экране истории и меню «...» в чате.

## Dynamic Island / Live Activity

`LumaWidgets/LumaAgentLiveActivity.swift` — использует тот же `LumaColor`
(включая монохромный `accent`) и переиспользует `Luma/DesignSystem`. Не
имитирует системное разворачивание Siri, не использует приватные жесты —
обычное системное нажатие открывает `luma://conversation/{id}`.
